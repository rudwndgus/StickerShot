# StickerShot (스티커샷)

> Snap it. Sticker it. Slice it. — 찍고, 모으고, 베어봐!

StickerShot은 휴대폰으로 물건을 찍어 투명 스티커로 저장하고, 스티커 통에서 실제 물리처럼 가지고 놀거나 손가락으로 베는 모바일 중심 PWA 웹게임입니다.

## 실행

Node.js 20 이상을 권장합니다.

```bash
npm install
npm run dev
```

프로덕션 빌드 및 테스트:

```bash
npm run test
npm run build
npm run preview
```

카메라, PWA 설치, 서비스 워커는 보안 컨텍스트가 필요합니다. 개발 중 `localhost`에서는 동작하며, 휴대폰 실기기 테스트에는 HTTPS 주소를 사용하세요.

## 주요 기능

- 카메라 촬영 또는 사진 보관함 선택
- 중앙 기준점과 외곽 배경 힌트를 사용하는 OpenCV GrabCut 기반 물체 분리
- 손바닥 위 물건 촬영 시 피부색 영역을 배경 힌트로 사용하는 손 인식 보정
- 원본 사진에서 실제 처리된 투명 PNG가 들리며 배경이 사라지는 스티커 떼기 전환
- 배경 제거 강도, 가장자리, 흰색 다이컷 테두리 편집
- 원본 전체 사진을 보관하지 않고 최적화한 투명 PNG와 썸네일만 IndexedDB에 저장
- Matter.js 중력·충돌·회전·드래그 기반 스티커 통
- iOS의 명시적 센서 권한 요청, 화면 방향 보정, dead zone과 low-pass smoothing
- 손가락 궤적과 회전 좌표 변환을 이용한 실제 두 조각 이미지 베기
- 점수, 콤보, 결과 화면
- standalone PWA, 오프라인 코드 캐시, 사용자 확인 후 업데이트

## iPhone에서 홈 화면에 추가

1. Safari에서 배포된 StickerShot 주소를 엽니다.
2. 공유 버튼을 누릅니다.
3. **홈 화면에 추가**를 선택합니다.
4. 홈 화면의 StickerShot 아이콘으로 실행합니다.

카메라 권한을 거부했거나 카메라가 없는 기기에서는 사진 보관함 선택 버튼을 이용할 수 있습니다. 기울기 센서는 `내 물건 보기` 화면의 버튼을 직접 눌렀을 때만 권한을 요청합니다.

## 이미지 처리 방식

모든 이미지 처리는 서버 업로드 없이 기기 안에서만 수행됩니다. 촬영 화면의 중앙 점을 확정 전경, 사진 외곽을 확정 배경으로 사용해 OpenCV GrabCut을 반복 실행합니다. 이후 중앙과 연결된 물체만 남기고 마스크의 작은 틈과 노이즈를 정리합니다. 출력 최대 변은 1280px, 분석 최대 변은 640px로 제한해 모바일 메모리를 제어합니다. 첫 스캔에서는 약 4MB(gzip)의 비전 엔진을 한 번 내려받아 기기 캐시에 보관합니다.

## 물리와 베기

스티커 통은 Matter.js world 하나를 사용합니다. 화면 밖 네 방향에 정적 경계를 두고, 스티커 썸네일에 맞춘 chamfer body에 중력·마찰·탄성·공기 저항을 적용합니다. 기울기 값은 화면 회전을 보정한 뒤 smoothing, dead zone, clamp를 거쳐 world 중력으로 변환됩니다.

게임은 칼날의 직전 선분과 회전된 물체 영역의 교차를 검사합니다. 교차한 선분을 물체 로컬 좌표로 역변환한 뒤, 두 개의 Canvas clip 마스크로 서로 다른 투명 텍스처를 만들고 각각 독립 Matter body로 교체합니다. 두 조각에는 절단선 법선의 반대 방향 충격과 회전을 줍니다. 임시 Canvas와 body는 화면 밖으로 나가면 제거합니다.

## IndexedDB와 업데이트

`stickershot-db`는 additive migration을 사용합니다. 버전 업그레이드 시 기존 object store를 지우지 않으며, 새 필드는 읽을 때 기본값을 보충합니다. 서비스 워커 업데이트는 `prompt` 방식입니다. 새 배포가 감지돼도 플레이 중 자동 새로고침하지 않고 하단 안내에서 사용자가 **지금 업데이트**를 선택할 때만 활성화합니다. 앱 코드 캐시 갱신은 IndexedDB 스티커 데이터를 삭제하지 않습니다.

## GitHub Pages 배포

`.github/workflows/deploy.yml`이 저장소 루트의 앱을 빌드해 Pages artifact로 배포합니다. 저장소 설정의 **Pages → Build and deployment → Source**를 **GitHub Actions**로 설정하세요. Vite는 상대 base를 사용해 프로젝트 하위 경로에서도 manifest, assets, service worker가 동작합니다.

## 알려진 제한

- 순수 온디바이스 색상 분리는 인물 머리카락이나 매우 복잡한 배경을 위한 대형 ML 모델보다 정확도가 낮습니다.
- iOS Safari는 진동 API를 지원하지 않아 진동은 사용하지 않습니다.
- 메모리 보호를 위해 물리 화면에는 최근 스티커 최대 22개가 활성화됩니다. 보유 개수는 제한하지 않습니다.
- 기기와 브라우저 정책에 따라 센서 권한을 다시 켜려면 Safari 사이트 설정을 변경해야 할 수 있습니다.
